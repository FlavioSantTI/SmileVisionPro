# SmileVision Pro

O **SmileVision Pro** é uma solução avançada de gestão inteligente de imagens clínicas, projetada para otimizar o fluxo de trabalho em consultórios odontológicos.

## Funcionalidades Principais

*   **Captura Clínica Inteligente:** Interface otimizada para captura de imagens de alta qualidade durante o atendimento.
*   **Gestão de Sessão:** Acompanhamento em tempo real do atendimento, com contexto claro de Clínica, Dentista e Paciente.
*   **Galeria de Sessão:** Visualização e gestão das capturas em tempo real, com suporte a ações em massa (seleção, limpeza).
*   **Sincronização de Dados:** Integração robusta para sincronização de imagens clínicas com o banco de dados.
*   **Exportação de Relatórios:**
    *   Geração de relatórios em PDF com layout profissional, mantendo a qualidade e proporção das imagens.
    *   Exportação de arquivos originais em formato ZIP, seguindo convenção de nomes baseada no paciente e data.

## Melhorias Recentes

*   **Redesign do Cabeçalho:** Layout moderno com breadcrumbs para contexto de sessão, hub de ações unificado e status de hardware compacto.
*   **Refinamento da Galeria:** Interface limpa, ícones intuitivos e ações em massa mais eficientes.
*   **Otimização de PDF:** Correção de proporção de imagens, layout em colunas, remoção de informações redundantes e adição de logo da clínica.
*   **Login e Perfil:** Personalização da página de login e ajustes na identificação do usuário.

## Tecnologias Utilizadas

*   **Frontend:** React (Vite), TypeScript, Tailwind CSS.
*   **Animações:** `motion` (framer-motion).
*   **PDF/ZIP:** `jsPDF`, `jszip`.
*   **Ícones:** `lucide-react`.

## Como subir para o GitHub

Para publicar este projeto no GitHub, siga os passos abaixo:

1.  **Crie um repositório** no GitHub.
2.  **Inicialize o Git** localmente (se ainda não estiver):
    ```bash
    git init
    ```
3.  **Adicione os arquivos**:
    ```bash
    git add .
    ```
4.  **Faça o commit inicial**:
    ```bash
    git commit -m "Initial commit: SmileVision Pro"
    ```
5.  **Adicione a URL do repositório remoto**:
    ```bash
    git remote add origin <URL_DO_SEU_REPOSITORIO>
    ```
6.  **Envie para o GitHub**:
    ```bash
    git push -u origin main
    ```
